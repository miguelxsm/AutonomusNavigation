PROJECT WORKFLOW
================

1. Which map do I actually use?
-------------------------------

You have two different kinds of map information:

- Reference geometry from the statement:
  - `statement/map.dxf`
  - `statement/map_*.png`

These are reference files for understanding the environment and for building a
future Gazebo world closer to the lab.

- Operational map used by Nav2 / SLAM:
  - generated live by `slam_toolbox`
  - saved as `.yaml` + `.pgm`

For the project workflow, the important map is the one generated by SLAM while
the robot explores.


2. How do I obtain the map the robot will traverse?
---------------------------------------------------

Local simulation:

1. Start the local validation stack.
2. Move the robot manually with teleop to explore.
3. Save the generated map.

Commands:

cd /home/miguel/Escritorio/ROB/project
./start_local_validation.sh

In another terminal:

cd /home/miguel/Escritorio/ROB/project
source source_local_env.sh
ros2 run turtlebot3_teleop teleop_keyboard

When the explored area looks good, save it:

cd /home/miguel/Escritorio/ROB/project
./save_current_map.sh sim_lab_map

This creates:
- `generated_maps/sim_lab_map.yaml`
- `generated_maps/sim_lab_map.pgm`


3. How do I run Gazebo and actually see the robot?
--------------------------------------------------

Fast GUI mode:

cd /home/miguel/Escritorio/ROB/project
./start_local_validation_gui.sh

This opens the local simulation with:
- Gazebo GUI
- SLAM Toolbox
- Nav2
- RViz

Then, in another terminal, move the robot:

cd /home/miguel/Escritorio/ROB/project
./teleop_local_nav.sh

And then test a goal:

cd /home/miguel/Escritorio/ROB/project
./send_relative_goal.sh 0.6 0.0 0.0

Why this wrapper matters:

- with Nav2 active, keyboard commands must go through `/cmd_vel_nav`
- Nav2 then smooths them and outputs `/cmd_vel`
- Gazebo listens on `/cmd_vel`


4. How do I know the project is really ready?
---------------------------------------------

Minimum ready checklist:

- simulator launches cleanly
- teleop moves the robot
- /scan, /odom, /tf, /map exist
- Nav2 reaches active state completely
- simple relative goal is accepted and executed
- SLAM map can be saved
- logs do not show repeated bringup failures

Project-level ready checklist:

- route to B / O / Base works in simulation
- CSV logging works
- obstacle detection works
- station detection works at least in controlled tests
- precision parking works in a simplified setup
